# Документация проекта

## Структура документации

- **`TROUBLESHOOTING.md`** - Библия ошибок и решений. **НЕПРИКОСНОВЕННЫЙ ДОКУМЕНТ.** Все ошибки и их решения должны быть задокументированы здесь.

## Как использовать документацию

1. **При возникновении ошибки:**
   - Сначала проверь `TROUBLESHOOTING.md`
   - Если похожая ошибка уже решена - используй существующее решение
   - Если нет - реши проблему и задокументируй решение в `TROUBLESHOOTING.md`

2. **При изменении кода:**
   - Проверь, не затрагивает ли изменение задокументированные решения
   - Если да - обнови документацию, сохранив старую версию

3. **Правила работы с документацией:**
   - НЕ удаляй разделы с решениями
   - НЕ изменяй решения без обновления документации
   - Всегда добавляй новые решения, не заменяя старые
